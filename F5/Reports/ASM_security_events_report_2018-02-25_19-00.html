Violations
Illegal parameter value length
Description
The system checks that the request contains a parameter whose value length (in bytes) matches the value length defined in the security policy.
Risk
Sending a parameter value length which the web application was not expecting to handle can result in various forms of buffer overflow attacks and denial of service attacks.
Attack Types
Abuse of Functionality
Examples
It is possible to restrict the length of a parameter value to be short (less than 20 characters). In this case, there is an additional value; this can help mitigate other injection attacks like XSS and SQL injection.
Note that this violation is relevant only for user input parameters. The ASM can be configured to block requests with parameters whose value exceeds a certain length.
